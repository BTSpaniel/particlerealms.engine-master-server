WebGPU OS Companion Privacy Policy

Effective and last updated: July 22, 2026

WebGPU OS Companion processes browser and AI data only to provide features the user requests. The developer does not operate an analytics or data-collection server for this extension, does not sell user data, and does not use user data for advertising.

What the extension processes

  • Web browsing activity: tab URLs, titles, navigation events, and limited in-memory history only after the user enables Browsing activity. This data is returned to the WebGPU OS interface and is not sent to the developer.
  • Website content: content selected for an explicit browser action, embedded-page compatibility, console messages from sites embedded in WebGPU OS, and screenshots requested by the user or an approved AI tool.
  • Audio: audio from a tab only after Tab audio is enabled and the OS mixer requests capture. Captured audio is processed and replayed locally; it is not sent to the developer.
  • Authentication information: API keys or an OpenRouter OAuth result that the user chooses to save. Credentials are encrypted with AES-GCM before local or session storage and are never exposed to the WebGPU OS page.
  • AI request data: prompts, selected page content, files, images, audio, model choices, and request metadata that the user submits through WebGPU OS.
  • Extension settings: capability choices, ad-protection state, counters, approved custom AI endpoints, and cached model metadata.

How data is used and shared

Browser data is used only for the visible browser, automation, protection, and audio features the user enables. AI request data is transmitted directly from the extension to the AI provider or custom endpoint selected by the user so that provider can return the requested result. The provider processes that request under its own terms and privacy policy. The developer does not receive these requests or responses.

The extension does not sell data, transfer data for advertising, use data for creditworthiness or lending, or permit humans to read user data except where the user gives specific consent for support, where required for security or law, or where data is aggregated and anonymized for internal operations.

Storage and retention

Session credentials are stored in chrome.storage.session and clear when the browser session ends. Persistent credentials are stored only when the user chooses “Remember new credentials.” Credential envelopes are encrypted locally; the non-extractable encryption key is held by the browser in IndexedDB. Other settings and model metadata are stored in extension-local storage. Navigation history is held only in service-worker memory, is limited to 200 entries, and clears when that context ends.

Users can remove credentials in the Secure AI Vault, disable any capability in the extension popup, clear extension storage through Chrome, or uninstall the extension to remove locally stored extension data.

Security and user control

Sensitive capabilities are off by default. The extension accepts privileged requests only from the exact production origins https://particlerealms.online and https://www.particlerealms.online, plus the explicit development origins http://localhost:9001 and http://127.0.0.1:9001. Other localhost ports are rejected. Browser JavaScript automation uses Chrome's User Scripts API and requires Chrome's separate “Allow User Scripts” control. Proxy requests reject private-network destinations and URL credentials.

Chrome Web Store Limited Use

The use of information received from Chrome APIs will adhere to the Chrome Web Store User Data Policy, including the Limited Use requirements.

Changes and contact

Material changes to data practices will be disclosed in the extension interface before the changed collection or use begins. Questions or privacy requests can be submitted to the project owner through github.com/BTSpaniel.
